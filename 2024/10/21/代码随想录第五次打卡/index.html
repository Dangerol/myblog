349. 两个数组的交集

常见的数组解法，考虑到数据量不大在0-1000之间，直接创建两个hash函数

用c++自带到hash数据结构，区别常见的set，multiset， unordered_set，map，multimap，unordered_map的区别和什么时候使用？

容器 有序性 元素唯一性 数据结构 适用场景
set 有序 唯一 红黑树 需要保持元素唯一且有序时
multiset 有序 不唯一 红黑树 元素可以重复且需要有序时
unordered_set 无序 唯一 哈希表 不需元素有序但需快速访问且保持元素唯一时
map 有序 键唯一 红黑树 需要根据唯一键快速检索数据且键有序时
multimap 有序 键不唯一 红黑树 同一个键可以对应多个值且需要键有序时
unordered_map 无序 键唯一 哈希表 键的唯一性和快速访问是主要需求，键的顺序不重要时 
•	有序 vs 无序：set、multiset、map 和 multimap 是有序的，维护元素的顺序；而 unordered_set 和 unordered_map 是无序的，提供更快的访问速度。
•	唯一 vs 非唯一：set 和 map 保持键或元素的唯一性；multiset 和 multimap 允许重复。
•	键值对 vs 单值：map 和 multimap 存储键值对，而 set 和 multiset 只存储单一元素。
